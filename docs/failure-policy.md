# 실패 처리 및 운영 정책

## 1. 자동 재시도 대상
- 파일 열기 실패
- JSONL 쓰기 실패
- 일부 에이전트 응답 누락
- 스키마 필드 누락
- 합의 계산 오류
- 엑셀/CSV 출력 생성 실패

## 2. 에스컬레이션 대상
- 입력 파일 자체가 비지원 형식일 때
- 다수 에이전트가 기준 자체를 다르게 해석할 때
- recall 과 precision 충돌이 너무 커서 정책 선택이 필요할 때
- QA가 실제 사용자 파이프라인 수행에 실패했을 때
- 필수 출력 파일을 구조적으로 생성할 수 없을 때

## 3. 스킵 + 로그 대상
- 일부 줄 손상
- 극소수 인코딩 이상 레코드
- 정규화 불가능한 소수 특수토큰

## 4. 합의 규칙
- 중요 의미 판정 단계에서는 최소 5개 독립 판정 확보
- 반대검토 단계를 생략하지 않는다
- reject 는 accept 보다 더 높은 증거 기준 요구
- 충돌이 크면 accept 또는 review / accept_with_risk 경로를 우선 검토
- 확신이 낮아도 후속 조합 단계 활용 가능성이 있으면 보존 쪽으로 기울인다
