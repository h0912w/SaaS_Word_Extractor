# QA 및 검증 기준

## 1. QA 핵심 원칙
- QA 전용 별도 소프트웨어를 만들지 않는다.
- QA는 실제 사용자와 동일한 진입점과 동일한 파이프라인을 실행한다.
- QA 결과도 단일 판단에 의존하지 않고 다중 에이전트가 검토한다.
- QA는 형식 검증 + 의미 검증 둘 다 포함한다.

## 2. 단계별 검증 기준
| 단계 | 성공 기준 | 검증 방법 | 실패 시 처리 |
|---|---|---|---|
| 입력 탐색 | 대상 파일 식별 완료 | 규칙 기반 + LLM 자기 검증 | 재시도 1회 후 에스컬레이션 |
| 로드/해제 | 로딩 성공, 줄 수 기록 | 규칙 기반 + 스키마 + LLM 자기 검증 | 재시도 1회 |
| 정규화 | 정규화 로그 포함 | 스키마 + 규칙 기반 + LLM 자기 검증 | 재시도 2회, 일부 스킵 |
| 규칙 스크리닝 | 명백한 노이즈 제거, 애매 케이스 보존 | 규칙 기반 + LLM 자기 검증 | 재시도 2회 |
| 1차 의미 판정 | 단어별 5개 독립 판정 확보 | 스키마 + LLM 자기 검증 | 재시도 1회 |
| 반대검토 | 단어별 3개 이상 반대/보완 의견 확보 | LLM 자기 검증 | 재시도 1회 |
| 재반박 | 충돌 사례마다 보완 근거 존재 | LLM 자기 검증 | 재시도 1회 |
| 최종 합의 | 결정/사유/합의 수치 완비 | 스키마 + 규칙 기반 + LLM 자기 검증 | 재시도 1회 |
| 결과 저장 | JSON 파싱 가능, 필수 파일 생성 | 스키마 + LLM 자기 검증 | 재시도 2회 |
| 사람용 출력 | XLSX/CSV 열람 가능, 컬럼/시트 구조 적합 | 스키마 + 규칙 기반 + LLM 자기 검증 | 재시도 2회 |
| QA 실행 | 실제 엔트리포인트 실행 완료 | 규칙 기반 + LLM 자기 검증 | 에스컬레이션 |
| QA 검토 | 치명적 결함 없음 | 스키마 + 규칙 기반 + LLM 자기 검증 + 사람 검토(선택) | 재시도/에스컬레이션 |

## 3. QA 샘플링 원칙
- 대용량 전체 입력 대신 샘플링 가능
- 단, 실사용 파이프라인은 동일해야 함
- 일반 단어 / 경계 사례 / 노이즈 사례 / 브랜드형 후보 / 기능형 후보 포함
- 샘플 추출 기준과 샘플 목록은 `/output/qa/` 에 기록

## 4. QA 승인 조건
- 출력 파일 생성 성공
- JSONL/JSON 무결성 이상 없음
- 경계 사례에서 과다 reject 징후 없음
- 명백한 노이즈 과다 유입 없음
- 다중 QA 검토 결과 치명적 결함 없음
